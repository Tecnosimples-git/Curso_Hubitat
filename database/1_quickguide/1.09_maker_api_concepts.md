# Capítulo 9: Conceitos Básicos Maker API

## Objetivo do Capítulo
Explicar de forma simples o que é a Maker API, por que ela é necessária para o VETRA, e os conceitos de segurança local.

## Roteiro de Gravação

1. **O que é a Maker API?**
   - Explicar o conceito de API de forma leiga: É uma ponte que permite que sistemas externos conversem com o Hubitat.
   - Mostrar que o Hubitat possui processamento 100% local, o que significa que o VETRA falará com ele sem precisar de nuvem. e caso cliente esteja fora de casa, ele tambem tem nuvem

2. **Como Funciona na Prática**
   - A Maker API expõe endpoints (URLs locais) para ligar, desligar ou ler o status dos dispositivos.
   - Explicar a segurança: Para acessar esses endpoints, é necessário um **Token de Acesso**, como se fosse uma senha invisível.

3. **Por que o VETRA precisa disso?**
   - O VETRA é uma interface (dashboard) que roda em tablets e celulares. Ele usa a Maker API para buscar os dispositivos e desenhar a tela, enviando comandos instantâneos de volta.

## Dicas para o Instrutor
- Não aprofunde em código JSON, foque apenas na analogia da "Ponte" e da "Senha" (Token). Isso tira o medo da complexidade.
