# Capítulo 10: Setup Maker API + VETRA

## Objetivo do Capítulo
Instalar e configurar a Maker API no Hubitat, selecionar os dispositivos que queremos expor, e finalmente conectá-los ao aplicativo VETRA para uso.

## Roteiro de Gravação

1. **Instalando a Maker API no Hubitat**
   - Navegar para **Apps -> Add Built-In App**.
   - Rolar a lista, encontrar e clicar em **Maker API**.
   - A tela de configuração da Maker API será aberta.

2. **Selecionando os Dispositivos na Maker API**
   - Explicar o conceito de segurança: O VETRA só verá os dispositivos que selecionarmos explicitamente nesta lista.
   - Detalhar a regra de seleção, pois cada arquitetura de integração exige uma seleção diferente:
     - **Módulos TCP (Relay e Dimmer):** Selecionar *apenas* os dispositivos **FILHOS** (ex: "MolSmart Relay - Switch 1"). Não selecione o Pai.
     - **Central de IR/RF (GW8):** Selecionar sempre o dispositivo **PAI** (o Virtual principal de Ar, TV ou RF).
     - **Áudio (SoundSmart):** Selecionar os dispositivos **PAI** (os Players e os Groups).
     - **Sensores e Módulos Zigbee:** Seguem a mesma lógica TCP. Para relés, dimmers e acionamentos em geral (com múltiplos canais), selecionar apenas os dispositivos **FILHOS**. Sensores simples vão direto.

3. **Opções da Maker API**
   - Habilitar as opções "Allow Access via Local IP address" e "Enable Event POSTing" (se o VETRA precisar de push).
   - Obter os dados de conexão que ficam na parte inferior da página:
     - URL Local (ex: `http://192.168.1.50/apps/api/100`)
     - Access Token (ex: `a1b2c3d4...`)
   - Clicar em "Done".

4. **Configurando o VETRA**
   - Mostrar a documentação oficial de integração na tela acessando **[ajuda.vetra.center](https://ajuda.vetra.center/)**.
   - Orientar os alunos a sempre seguirem o passo a passo oficial descrito lá para inserir a URL e o Token, garantindo que estejam vendo a versão mais atualizada da interface do VETRA.

5. **Importando seu projeto para o VETRA**
   - Acessar a tela de importação no app VETRA e explicar os 3 métodos disponíveis:
     - **Importação Total:** Sincroniza todos os dispositivos de uma vez.
     - **Importação por Ambientes:** Importa dispositivos agrupados por cômodos.
     - **Importação por Devices:** Permite pinçar equipamentos específicos avulsos.
   - **Regra de Ouro:** Reforçar que, para a importação por ambientes funcionar de forma automática, os *Rooms* já precisam estar criados e os dispositivos vinculados a eles previamente no Hubitat.
   - Realizar *imports* separados na prática para demonstrar a flexibilidade do sistema e o resultado na tela.

6. **Personalização de Uso (Conferência e Ajuste de Tipos)**
   - Após a importação, o passo fundamental é navegar pelo VETRA e conferir se todos os dispositivos foram adicionados corretamente.
   - **Troca de Tipo:** Demonstrar detalhadamente como alterar o *Tipo de Dispositivo* diretamente no VETRA. Explicar que, muitas vezes, o Hubitat manda um "Interruptor Genérico", mas na prática aquele canal pode ser um switch que aciona cena, e não uma iluminação, ou sensor de contato que ele reconheceu como swithc. Ajustar o tipo garante o ícone correto e o comportamento ideal na interface.

7. **Primeiro Teste de Controle**
   - Retornar à tela inicial do VETRA.
   - Tocar em um botão do VETRA e mostrar a lâmpada/equipamento acionando instantaneamente no cenário real.

## Dicas para o Instrutor
- Esta é a "grande revelação" do curso, onde todo o trabalho de backend resulta numa interface visual final e bonita. Demonstre entusiasmo ao exibir o dashboard no VETRA e controlar os itens pela primeira vez!
