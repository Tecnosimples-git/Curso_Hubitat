# Capítulo 2: Conceitos Básicos e Familiaridade com Telas

## Objetivo do Capítulo
Familiarizar o aluno com a interface de usuário do Hubitat, explicando para que serve cada menu principal, sem entrar em configurações complexas.

## Roteiro de Gravação

1. **Apresentação do Menu Principal (Esquerda)**
   - Mostrar a barra de navegação lateral. Explicar que ela é o coração da navegação do sistema e passar rapidamente por cada item.

2. **Rooms (Cômodos)**
   - Explicar que serve para agrupar e organizar os dispositivos fisicamente (ex: Sala de Estar, Quarto Principal), facilitando a gestão da casa.

3. **Dashboards**
   - Mencionar que o Hubitat possui dashboards nativos para controle rápido, mas dar o *spoiler* de que usaremos o VETRA no final do curso para ter uma interface visual premium.

4. **Devices (Dispositivos)**
   - Explicar que este é o menu mais usado. Aqui ficam listados todos os equipamentos físicos e virtuais (Lâmpadas, Relés, Sensores, TVs).
   - Mostrar a tela de detalhes de um dispositivo (Device Page) com os comandos, preferências e estados (Current States).

5. **Apps (Aplicativos)**
   - Explicar o conceito: "Apps adicionam inteligência e conectividade". Eles gerenciam pacotes e conectores essenciais (como o Maker API e o Hubitat Package Manager).

6. **Integrations (Integrações)**
   - Mostrar que este menu agrupa conexões com serviços de terceiros e assistentes de voz (como Amazon Alexa, Google Home, Apple HomeKit).

7. **Automations (Automações)**
   - Explicar que é aqui que a "mágica" acontece. É o lar dos aplicativos nativos de regras lógicas, como o *Basic Rules* e o poderoso *Rule Machine*.

8. **Subscriptions (Assinaturas)**
   - Mencionar brevemente os serviços premium opcionais oferecidos pela Hubitat (como Hubitat Protect para backup na nuvem e Remote Admin para acesso fora de casa).

9. **Settings (Configurações)**
   - Mostrar as opções essenciais do sistema: configurações de rede, reboot/shutdown seguro do hub e a seção de Backups locais.

10. **Logs**
    - Destacar a importância desta tela como o "raio-x" do sistema para diagnóstico. Mostrar a diferença entre monitoramento em tempo real (*Live Logs*) e o histórico (*Past Logs*).

11. **O que precisa alterar no primeiro setup?**
- fusorario (time zone, date format, time format)
- developer tools
- hub login security
- zigbee details (canal 15,20,25, power level mantem padrao)
- zwave details (enable disable, frequencia AU)

## Dicas para o Instrutor
- Use analogias: "Devices são os músculos, Automations e Apps são o cérebro".
- Mantenha o tom encorajador. A interface tem muitas opções, mas ressalte que, no dia a dia, eles usarão apenas 2 ou 3 menus com mais frequência.
