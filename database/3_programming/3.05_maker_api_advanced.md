# Módulo 3 - Programação e Lógica: Maker API (Avançado)

## Objetivo da Aula
O aluno descobrirá as entranhas da integração LAN e Cloud do Hubitat, utilizando a Maker API para disparar requisições HTTP REST (GET/POST) e receber Webhooks, permitindo conectar o Hubitat a hardwares que não possuem drivers nativos.

## Roteiro de Gravação

1. **A Filosofia Maker API (Documentação Oficial)**
   - Relembrar o Módulo 1: O VETRA usa a Maker API lendo os URLs do Hubitat (Hubitat como Servidor).
   - Mostrar que a Maker API possui URLs tanto para rede Local (LAN) quanto via Cloud (https://cloud.hubitat.com/...).

2. **Disparando Comandos via URL (Endpoint de Ação)**
   - Explicar a estrutura oficial de envio de comandos detalhada na página do app:
     `http://[IP-do-Hub]/apps/api/[App-ID]/devices/[Device-ID]/[Command]/[Secondary-Value]?access_token=[Token]`
   - Demonstrar o disparo dessa URL diretamente no navegador e mostrar a lâmpada acendendo.

3. **Postar Eventos de Dispositivos (Event POSTing)**
   - Mostrar a configuração *URL to send device events to by POST* dentro da Maker API.
   - Explicar que isso envia um JSON automático para um servidor externo (como o VETRA ou um servidor Node.js) toda vez que um sensor muda de estado.

4. **Rule Machine: Mandando Requisições (Send HTTP Request)**
   - Como o Hubitat age como Cliente para comandar outro sistema?
   - Abrir o Rule Machine. Nas *Actions*, procurar por `Network -> Send HTTP Request`.
   - Configurar um envio de método `POST` para um endpoint externo (ex: IFTTT ou API de uma TV moderna). Abordar a inserção de JSON na área de *Body*.

## Dicas para o Instrutor
- Esta aula consagra o aluno como um integrador sênior. Saber trabalhar com as URLs de comando da Maker API e enviar *HTTP Requests* pelo Rule Machine significa que o aluno pode integrar qualquer API aberta do mercado mundial no Hubitat.
