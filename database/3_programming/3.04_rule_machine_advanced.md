# Módulo 3 - Programação e Lógica: Rule Machine (Avançado)

## Objetivo da Aula
Elevar o nível do integrador, ensinando comandos complexos do **Rule 5.1** extraídos da documentação: blocos condicionais IF-THEN, esperas baseadas em eventos (Wait for Event) e a poderosa funcionalidade global de *Hub Variables*.

## Roteiro de Gravação

1. **Condicionais (IF-THEN-ELSE)**
   - Partir da regra básica e adicionar lógica interna nas *Actions*.
   - Inserir um bloco estrutural IF: `IF (Time is between Sunset and Sunrise) THEN`
   - Adicionar ações apenas para aquele bloco (ex: Acender luz a 10%).
   - Mostrar a importância vital de fechar o bloco lógico com a ação `END-IF`.

2. **Wait for Events vs Delays (Dica Crucial)**
   - O manual do Hubitat diferencia muito isso:
   - **Delay Action:** Pausa cega. O hub apenas espera "X segundos" e continua.
   - **Wait For Event:** Pausa inteligente. A regra congela até que um dispositivo responda. Ex: Mandar fechar uma cortina e, usar `WAIT FOR Contact Sensor Closed`, para só então mandar uma notificação de sucesso.

3. **Repeats (Laços de Repetição)**
   - Mostrar a ação *Repeat Actions*. Excelente para piscar luzes como alarme ou checar a temperatura a cada 5 minutos até o ar condicionado estabilizar. Lembrar sempre de usar a ação *Stop Repeating Actions* no final.

4. **Hub Variables (O "Cérebro" Global)**
   - Explicar que *Hub Variables* substituíram as antigas Globais. Elas ficam na aba **Settings -> Hub Variables**.
   - Criar uma variável booleana chamada `Modo_Festa`.
   - Mostrar como colocar essa variável exposta num Dashboard VETRA através de um *Connector*.
   - Voltar no Rule Machine e inserir a condição "Apenas execute se `Modo_Festa` for FALSO".

## Dicas para o Instrutor
- Cuidado com o "Wait for Event". Alerte os alunos que se o evento nunca ocorrer (o sensor quebrou), a regra fica congelada aguardando. Ensine a colocar um "Timeout" no Wait, conforme recomendado nas boas práticas do Rule Machine.
