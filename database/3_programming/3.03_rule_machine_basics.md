# Módulo 3 - Programação e Lógica: Rule Machine (Básico)

## Objetivo da Aula
Introduzir o aplicativo de regras mais avançado do ecossistema: O **Rule Machine (Rule 5.1)**. Baseado na documentação oficial, o aluno aprenderá a lógica fundamental (Triggers, Conditions e Actions) para criar lógicas If-This-Then-That sem escrever código Groovy.

## Roteiro de Gravação

1. **Apresentação: O que é o Rule 5.1?**
   - Explicar que o Rule Machine permite automatizar qualquer coisa que não seja exclusivamente iluminação (onde o Room Lighting atua melhor).
   - Criar uma nova regra no app.

2. **A Tríade Lógica (Documentação Oficial)**
   - **Trigger Events:** Eventos que acionam a execução de uma regra. Eles *ouvem* mudanças no sistema (Ex: Porta abre, Botão é pressionado, Hora exata do dia).
   - **Conditions (Condições):** Opcionais. São estados que a regra avalia (Verdadeiro ou Falso) para tomar decisões *dentro* das ações.
   - **Actions to Run:** As tarefas que o Hubitat executará em sequência, de cima para baixo.

3. **Criando a Regra de Exemplo**
   - **Caso prático:** Ligar o amplificador SoundSmart e uma luz quando um botão físico for apertado.
   - **Trigger:** Configurar para *Button Device* -> escolher o botão e a ação *pushed*.
   - **Actions:** Navegar pelas extensas categorias de ações do Hubitat.
     - `Control Switches/Push Buttons` -> Selecionar lâmpadas -> `Turn On`.
     - `Control Music Players` -> Selecionar o SoundSmart Player -> `Play`.

4. **Testando a Execução Linear**
   - Destacar o botão **Run Actions** na interface do Rule Machine, excelente dica oficial para testar se a lógica das ações funciona antes mesmo do Trigger ocorrer na vida real.
   - Apertar o botão e ver o hardware real respondendo.

## Dicas para o Instrutor
- Desmistifique o visual do Rule Machine. Ele assusta iniciantes. Reforce que se o aluno entender a separação clara entre "O QUE DISPARA A REGRA" (Trigger) e "O QUE A REGRA FAZ" (Action), ele domina a ferramenta.
